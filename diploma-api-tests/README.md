# Автотесты API

Дипломная работа по автоматизации **функционального** и **регрессионного** тестирования **бекенд части** сервиса управления задачами на примере Wekan.

## Что есть что

- **Функциональные тесты** — проверяют, что конкретные функции/сценарии работают согласно ожиданиям (требованиям). Это не означает, что они обязаны «задеть все функции кода» — в реальности тестируют выбранные требования и критические пользовательские сценарии.
- **Регрессионные тесты** — это стабильный набор проверок, который регулярно прогоняется после изменений, чтобы убедиться, что старое не сломалось. Обычно это отобранная часть функциональных тестов (ядро).
- **Smoke** — минимальный быстрый набор «сервис вообще жив?». Часто smoke-тесты входят в регрессию как самый быстрый слой.

## Структура проекта

- `diploma_tests/` — «библиотека» проекта: конфиг и HTTP-клиент для API
	- `config.py` — чтение настроек из `.env`/переменных окружения (BASE_URL, логин, пароль, таймауты)
	- `client.py` — обёртка над REST API (логин, создание/удаление сущностей и т.д.)
- `tests/` — сами тесты (pytest)
	- `smoke/` — быстрые проверки «живости» (1–3 E2E сценария)
	- `regression/` — регрессионный набор (стабильные проверки ключевых функций)
	- `functional/` — место под расширенные функциональные тесты (глубже/шире, чем регрессия)
	- `conftest.py` — фикстуры pytest (инициализация клиента, чтение настроек)
- `TEST_SUITE.md` — план «ядра регрессии» с привязкой к эндпоинтам

## Предусловия
- Запущенный Wekan (Docker). Обычно: `http://localhost/`
- Python 3.11+ 

## Установка
```powershell
cd C:\dev\mai\diploma\Bachelor\diploma-api-tests

python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Copy-Item .env.example .env
# Откройте .env и задайте WEKAN_USERNAME/WEKAN_EMAIL + WEKAN_PASSWORD
```

## Запуск
```powershell
# Все тесты
pytest -q

# Только smoke
pytest -m smoke -q

# Только regression
pytest -m regression -q

# Только functional
pytest -m functional -q
```

## Примечания
- Тесты создают временные доски/списки/карточки и стараются чистить за собой.
- Авторизация: `POST /users/login`, затем заголовок `Authorization: Bearer <token>`.
