# Проверки API 

Маркировка проверок:

- `smoke` — быстрые базовые проверки работоспособности.
- `regression` — стабильный набор по ключевым операциям.
- `functional` — расширенные проверки (негативные сценарии, граничные случаи, доступы).
- `slow` — длительные проверки, которые можно исключать из обычных прогонов.

## Что проверяется

Проверки работают только через REST API. В область входят:

- вход и запрет доступа без авторизации;
- операции по основным сущностям: доски, списки, карточки, дорожки;
- обработка ошибок (в том числе ситуация «HTTP 200 + ошибка в JSON»);
- устойчивость прогонов на локальном стенде.

В область не входят UI‑проверки и нагрузочные испытания.

## Конфигурация (.env)

Обязательные параметры:

- `BASE_URL`
- `WEKAN_USERNAME` или `WEKAN_EMAIL`
- `WEKAN_PASSWORD`
- `REQUEST_TIMEOUT_SECONDS`

Опционально (если нужны проверки с двумя пользователями):

- `WEKAN_USERNAME_2` или `WEKAN_EMAIL_2`
- `WEKAN_PASSWORD_2`

## Запуск

```powershell
# Все проверки
pytest -q

# Только быстрые
pytest -m smoke -q

# Регрессия
pytest -m regression -q

# Функциональные
pytest -m functional -q
```

## Структура

- `diploma_tests/` — общая библиотека проекта
  - `config.py` — чтение настроек из окружения/`.env`
  - `client.py` — клиент‑обёртка над REST API (CRUD по сущностям)
  - `http_helpers.py` — сетевые повторы и проверка «unauthorized» ответов
  - `waiters.py` — ожидания/polling для подтверждения удаления
- `tests/` — тесты
  - `smoke/` — smoke сценарии, которые быстро проверяют стенд
  - `regression/` — регрессионные проверки по ключевым операциям
  - `functional/` — расширенные проверки (часть уже реализована, часть оставлена как задел)


### Smoke 

Файл: `tests/smoke/test_smoke_flow.py`

- успешный вход и проверка, что токен подходит для защищённого запроса;
- запрет доступа без авторизации;
- создание доски, получение дорожек;
- создание списка, чтение списков;
- создание карточки, чтение карточек по дорожке;
- обновление карточки (заголовок) и проверка сохранения;
- перемещение карточки между списками;
- удаление карточки с подтверждением через polling;
- best‑effort очистка данных.

### Regression

Аутентификация и запрет доступа:

- `tests/regression/test_regression_auth.py`
  - вход с неверным/пустым паролем, отсутствие токена;
  - отказ без авторизации и с неверным токеном на защищённых эндпоинтах.

Доски:

- `tests/regression/test_regression_boards.py`
  - создание доски (контракт ответа);
  - видимость доски в списке;
  - чтение доски по id;
  - удаление доски и подтверждение отсутствия.

Списки:

- `tests/regression/test_regression_lists.py`
  - создание списка (возвращается id);
  - видимость списка в `GET /lists`;
  - чтение списка по id;
  - удаление списка и подтверждение отсутствия.

### Functional

Реализовано:

- `tests/functional/test_functional_auth.py`
  - успешный вход (по username/email) и проверка токена;
  - негативные случаи по учётным данным;
  - отказ на защищённом эндпоинте при неверном/пустом `Authorization`;
  - вход вторым пользователем (если настроен).
- `tests/functional/test_functional_boards.py`
  - контракт и видимость созданной доски;
  - негативный сценарий на создание с неверным набором полей;
  - проверка, что защищённые эндпоинты требуют авторизацию;
  - проверки доступа между пользователями (если настроены два пользователя);
  - добавление/удаление участника доски (если доступно на стенде);
  - запрос смены роли участника;
  - повторное удаление доски (поведение «как идемпотентное»).
- `tests/functional/test_functional_lists.py`
  - проверки авторизации для эндпоинтов списков;
  - контракт: создать → получить → удалить;
  - негативный сценарий: создание списка без title;
  - проверки доступа ко спискам вторым пользователем (если настроен).

## Ядро регрессии (15 сценариев)

Эти сценарии используются как обязательный минимум регрессии.

1) Успешный вход пользователя (`POST /users/login`).
2) Вход с неверным паролем (`POST /users/login`).
3) Запрет доступа без авторизации.
4) Создание доски (`POST /api/boards`).
5) Удаление доски (`DELETE /api/boards/{boardId}`).
6) Видимость доски в списке досок (`GET /api/users/{userId}/boards`).
7) Дорожки по умолчанию (`GET /api/boards/{boardId}/swimlanes`).
8) Создание списка (`POST /api/boards/{boardId}/lists`).
9) Видимость списка (`GET /api/boards/{boardId}/lists`).
10) Удаление списка (`DELETE /api/boards/{boardId}/lists/{listId}`).
11) Создание карточки (`POST /api/boards/{boardId}/lists/{listId}/cards`).
12) Видимость карточки по дорожке (`GET /api/boards/{boardId}/swimlanes/{swimlaneId}/cards`).
13) Обновление карточки (`PUT /api/boards/{boardId}/lists/{listId}/cards/{cardId}`).
14) Перемещение карточки (`PUT /api/boards/{boardId}/lists/{fromListId}/cards/{cardId}` + новый `listId`).
15) Удаление карточки (`DELETE /api/boards/{boardId}/lists/{listId}/cards/{cardId}`).
